# Change: add-chatgpt-github-sync-extension

## Why
当前缺少一个可持续、可审计的 ChatGPT 聊天记录沉淀机制。仅依赖人工复制会导致信息遗漏、无法追踪增量、难以在团队内复用。

目标是构建浏览器插件，支持手动与自动同步聊天记录到 GitHub，并保证同步结果可直接在 GitHub 中可视化浏览，且保留原始内容用于追溯。

## What Changes
- 新增浏览器插件同步能力，覆盖：
  - 手动同步：支持用户选择会话与目标分类目录后执行同步。
  - 自动同步：定时增量抓取会话新增内容并同步，周期与周期范围均可配置。
- 新增内容标准化能力，统一处理多类型消息（文本、代码块、Markdown、链接、图片/附件引用等）。
- 新增分类能力：
  - 手动分类优先（必备）。
  - 自动分类作为可插拔增强能力（非必备），低置信度时回退手动确认。
- 新增双轨输出：
  - 原始会话（raw）完整留存。
  - 分类后会话（categorized）按角色与业务组织。
- 新增 GitHub 可视化输出规范：
  - 结构化目录。
  - Markdown 导航页与索引。
  - 可回放的时间线视图。
- 新增同步可靠性和审计要求：
  - 增量游标、幂等写入、前 24 小时窗口去重、失败日志与关键失败告警。

## Impact
- Affected specs:
  - `session-sync`
  - `content-classification`
  - `github-visualization-output`
- Affected code (planned):
  - 浏览器插件：popup/options/content/background 同步链路
  - 领域层：会话抓取、标准化、分类、渲染、GitHub 提交
  - 基础设施：GitHub API 客户端、调度器、本地状态存储、日志与告警

## Assumptions
- 首期目标站点为 ChatGPT Web，会话内容以页面可读取数据为准。
- 首期支持 GitHub 仓库写入，并同时支持 GitHub App / OAuth / PAT 三种鉴权方式。
- “任何类型内容”在首期定义为：文本、代码块、Markdown、链接、图片/附件引用元数据；二进制原文件下载不作为首期强制范围。
- 自动同步周期与可配置范围由用户在设置中指定，系统需校验范围合法性。
- “业务”不采用固定枚举，按业务主题与进展情况进行动态标签化。
- 同步流程不强制要求专用分支或 PR，可直接发布到用户指定目标分支。
